# weeslee-rag Documentation Rules Skill

## 1. 목적

이 문서는 `weeslee-rag` 프로젝트의 docs 폴더에 작성되는 모든 문서의 형식 규칙을 정의한다.

문서 작성 시 일관된 명명 규칙과 구조를 유지하여:
- 작성 시점과 작성자를 명확히 추적
- 문서 검색 및 정렬 용이성 확보
- 여러 작성자 간 협업 시 혼란 방지

---

## 2. 파일명 규칙

### 2.1 기본 형식

```
YYYY-MM-DD_{작성자}_{문서제목}.md
```

### 2.2 구성 요소

1. **날짜 (YYYY-MM-DD)**
   - ISO 8601 형식 사용
   - 4자리 연도 + 2자리 월 + 2자리 일
   - 하이픈(`-`)으로 구분
   - 예: `2026-07-02`

2. **작성자**
   - 언더스코어(`_`)로 앞뒤 구분
   - 가능한 값:
     - `Claude`: Claude Code가 작성한 문서
     - `Codex`: GitHub Copilot Codex가 작성한 문서
     - `Lee`: 개발자(이동현)가 직접 작성한 문서
     - `ChatGPT`: ChatGPT가 작성한 문서
     - 기타 AI 모델명 또는 개발자명

3. **문서 제목**
   - 한글 사용 가능
   - 언더스코어(`_`)로 단어 구분
   - 공백 대신 언더스코어 사용
   - 특수문자 사용 최소화
   - 의미를 명확히 전달하는 제목 사용

### 2.3 파일명 예시

**Good Examples**:
```
2026-07-02_Claude_Dataset_Builder_Step9_Wiki_생성_문제점_및_개선사항.md
2026-07-02_Codex_ActiveSnapshot_시스템설정_DB통일_설계문서.md
2026-07-02_Lee_DatasetBuilder_단계재구성질문_검토결과.md
```

**Bad Examples**:
```
dataset_build_analysis_20260702.md          # 날짜 형식 위반, 작성자 누락
20260702_wiki_문제점.md                     # 날짜 구분자 누락, 작성자 누락
Wiki 생성 문제점.md                          # 날짜 누락, 작성자 누락, 공백 사용
```

## 3. 문서 구조 규칙

### 3.1 필수 메타데이터 (문서 상단)

```markdown
# {문서 제목}

**작성일**: YYYY-MM-DD
**작성자**: {작성자명}
**대상/범위**: {분석 대상이나 적용 범위}

---
```

### 3.2 문서 하단 푸터

```markdown
---

**작성자**: {작성자명}
**최종 수정**: YYYY-MM-DD
```

## 4. 문서 유형별 구조

### 4.1 문제점 분석 문서

- 현황 분석
- 문제점 상세 ([우선순위] 표기)
- 개선사항
- 우선순위 및 작업 순서
- 검증 계획
- 관련 파일

### 4.2 설계 문서

- 개요
- 요구사항
- 설계 (데이터 구조, API 명세, 처리 흐름)
- 구현 계획
- 검증 계획

---

## 5. 작성 규칙

### 5.1 우선순위 표기

- `[Critical]`: 즉시 수정 필요
- `[High]`: 단기 내 수정 필요
- `[Medium]`: 중기 개선 사항
- `[Low]`: 장기 개선 사항

### 5.2 상태 표기

- ✅ 완료/정상
- ⚠️ 경고/주의
- ❌ 실패/문제
- 🔄 진행 중
