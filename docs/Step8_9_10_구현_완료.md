## 구현된 API

### Step 8: Graph Build

Knowledge Graph를 빌드하는 API입니다.

#### 엔드포인트
- `POST /api/admin/dataset-builder/step8/build` - 그래프 빌드 실행
- `GET /api/admin/dataset-builder/step8/status` - 그래프 상태 조회
- `GET /api/admin/dataset-builder/step8/stats` - 그래프 통계 조회

#### 주요 기능
- 기존 `graph.py`의 `build_graph_jsonl.py` 스크립트를 호출
- source_id별 그래프 빌드 지원
- 노드/엣지 카운트 및 타입별 통계 제공

#### Request 예시
```json
{
  "source_id": null,
  "rebuild": false
}
```

#### Response 예시
```json
{
  "success": true,
  "message": "Graph built successfully: 150 nodes, 300 edges",
  "node_count": 150,
  "edge_count": 300,
  "processing_time": 25.3,
  "output": "..."
}
```

### Step 9: Wiki Build

프로젝트, 조직, 기술별 Wiki 문서를 자동 생성하는 API입니다.

#### 엔드포인트
- `POST /api/admin/dataset-builder/step9/build` - Wiki 빌드 실행
- `GET /api/admin/dataset-builder/step9/status` - Wiki 상태 조회
- `GET /api/admin/dataset-builder/step9/stats` - Wiki 통계 조회
- `GET /api/admin/dataset-builder/step9/list` - 생성된 Wiki 목록 조회

#### 주요 기능
- wiki_type 선택: `project`, `organization`, `technology`
- 기존 `wiki.py`의 빌드 스크립트 호출
- source_id별 Wiki 생성 지원
- 특정 slug만 생성 가능

#### Request 예시
```json
{
  "source_id": null,
  "wiki_type": "project",
  "slug": null,
  "from_inventory": false,
  "max_wikis": 0
}
```

#### Response 예시
```json
{
  "success": true,
  "message": "Wiki build completed: 25 wikis generated",
  "generated_count": 25,
  "generated_wikis": ["project-a", "project-b", ...],
  "processing_time": 180.5,
  "output": "..."
}
```

### Step 10: Search Quality

RAG 검색 품질을 테스트하고 평가하는 API입니다.

#### 엔드포인트
- `POST /api/admin/dataset-builder/step10/test` - 검색 품질 테스트 실행
- `GET /api/admin/dataset-builder/step10/status` - 검색 시스템 상태 조회
- `GET /api/admin/dataset-builder/step10/stats` - 검색 시스템 통계 조회
- `GET /api/admin/dataset-builder/step10/sample-queries` - 샘플 테스트 쿼리 조회

#### 주요 기능
- FAISS 벡터 검색 또는 GraphRAG Agent 선택 가능
- 여러 테스트 쿼리 일괄 실행
- Precision/Recall 자동 계산 (expected_doc_ids 제공 시)
- 평균 검색 시간 측정

#### Request 예시
```json
{
  "test_queries": [
    {
      "query": "ISP 방법론이 적용된 프로젝트는?",
      "expected_doc_ids": [101, 105, 112],
      "category": "proposal"
    },
    {
      "query": "클라우드 마이그레이션 관련 문서",
      "expected_doc_ids": null,
      "category": null
    }
  ],
  "source_id": null,
  "top_k": 10,
  "use_graph": false
}
```

#### Response 예시
```json
{
  "success": true,
  "total_queries": 2,
  "passed_queries": 2,
  "failed_queries": 0,
  "avg_precision": 0.67,
  "avg_recall": 0.80,
  "avg_search_time_ms": 125.5,
  "test_results": [
    {
      "query": "ISP 방법론이 적용된 프로젝트는?",
      "success": true,
      "results_count": 10,
      "results": [...],
      "precision": 0.60,
      "recall": 0.80,
      "search_time_ms": 120.3,
      "error": null
    },
    ...
  ]
}
```

## 기술적 세부사항

### Step 8 구현 특징
- `build_graph_jsonl.py` 스크립트를 subprocess로 실행
- 300초 타임아웃 설정
- JSON 출력 파싱하여 노드/엣지 카운트 추출
- 기존 graph.py의 `_load_graph()`, `_manifest()` 함수 재사용

### Step 9 구현 특징
- wiki_type에 따라 다른 빌드 방식 선택
  - `project`: `build_project_wiki.py` 스크립트 실행
  - `organization`: wiki.py의 `generate_wiki_by_organization()` 호출
  - `technology`: wiki.py의 `generate_wiki_by_technology()` 호출
- 600초 타임아웃 (프로젝트 Wiki 빌드는 시간이 오래 걸릴 수 있음)

### Step 10 구현 특징
- FAISS 검색과 GraphRAG Agent 선택 가능
- Precision/Recall 계산 로직 구현
- 여러 테스트 쿼리 일괄 처리
- 평균 성능 지표 계산
- 샘플 쿼리 제공 기능
