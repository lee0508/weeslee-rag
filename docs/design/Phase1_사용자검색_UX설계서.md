# Phase 1. 사용자 검색 UX 설계서

## 1. 문서 개요

이 문서는 `rag-assistant.html`의 사용자 검색 UX 흐름과 화면 구성을 정의한다.
사용자 화면에서 필요한 데이터를 먼저 정의하고, admin.html 데이터 생성 기능을 역산하는 기준 문서다.

## 2. 사용자 검색 시나리오

### 2.1 기본 검색 시나리오

```
1. 사용자가 질문을 입력한다
2. 시스템이 프롬프트를 분석한다 (intent, keywords, filters)
3. 4가지 검색 모드가 병렬로 실행된다
   - RAG: FAISS Vector 검색
   - RAG Agent: 전략적 도구 선택 검색
   - Graph RAG: 관계 기반 그래프 탐색
   - LLM Wiki: 사전 생성된 위키 검색
4. 검색 결과가 4개 탭에 표시된다
5. 통합 추천 문서 리스트가 상단에 표시된다
6. 사용자가 문서를 선택하면 상세 패널이 열린다
7. 사용자가 문서를 선택 후 "답변 생성"을 요청한다
8. 선택 문서만 context로 grounded answer가 생성된다
```

### 2.2 검색 모드별 특성

| 모드 | 용도 | 강점 |
| --- | --- | --- |
| RAG | 유사 문장/청크 검색 | 정확한 텍스트 매칭 |
| RAG Agent | 복합 질문 처리 | 도구 조합, 전략 선택 |
| Graph RAG | 관계 탐색 | 기관-프로젝트-기술 연결 |
| LLM Wiki | 사전 요약 검색 | 빠른 개요 파악 |

### 3.2 Left Sidebar (320px)

#### 검색 상태 카드
- 총 문서 수
- 인덱싱 상태 (ready/building/error)
- 마지막 업데이트 시간

#### 검색 설정
- 검색 모드 선택 (all, rag, agent, graph, wiki)
- Top K 결과 수 (5, 10, 20)
- 최소 점수 임계값 (0.0 ~ 1.0)
- 연도 필터 (2020~2026)
- 문서 유형 필터 (rfp, proposal, report, presentation)

#### 검색 결과 리스트
- 문서 카드 목록
- 각 카드: 제목, 점수, 유형, 발주기관, 수행연도
- 클릭 시 상세 패널 열림

## 4. 검색 모드 및 필터 정의

### 4.1 검색 모드

| 모드 | API | 설명 |
| --- | --- | --- |
| all | POST /api/search/all | 4개 모드 병렬 실행 |
| rag | POST /api/search/rag | FAISS Vector 검색 |
| agent | POST /api/search/rag-agent | 전략적 도구 검색 |
| graph | POST /api/search/graph-rag | 관계 그래프 탐색 |
| wiki | POST /api/search/llm-wiki | 위키 문서 검색 |

### 4.2 필터 타입

| 필터 | 타입 | 값 예시 |
| --- | --- | --- |
| year | select | 2020, 2021, 2022, 2023, 2024, 2025 |
| organization | autocomplete | K-water, LH, 한전, 국토부 |
| document_type | multiselect | rfp, proposal, report, presentation |
| technology_tags | chips | AI, 클라우드, 데이터플랫폼, IoT |
| business_tags | chips | ISP, ISMP, 컨설팅, SI, PMO |
| min_score | slider | 0.0 ~ 1.0 |
| top_k | select | 5, 10, 20, 50 |

## 5. 사용자 인터랙션 흐름

### 5.1 검색 → 결과 → 상세 → 답변 생성

```
[질문 입력]
    │
    ▼
[검색 버튼 클릭]
    │
    ├──▶ POST /api/rag-assistant/analyze-prompt
    │         └ intent, keywords, filters 반환
    │
    ├──▶ POST /api/search/rag (병렬)
    ├──▶ POST /api/search/rag-agent (병렬)
    ├──▶ POST /api/search/graph-rag (병렬)
    └──▶ POST /api/search/llm-wiki (병렬)
              │
              ▼
[검색 결과 표시]
    │
    ▼
[문서 카드 클릭]
    │
    ├──▶ GET /api/documents/{id}
    ├──▶ GET /api/documents/{id}/html
    └──▶ GET /api/documents/{id}/summary
              │
              ▼
[문서 상세 패널 표시]
    │
    ▼
[문서 선택 체크박스]
    │
    ▼
[답변 생성 버튼 클릭]
    │
    └──▶ POST /api/rag-assistant/generate-answer
              └ selected_docs, query, answer_mode: grounded_only
                    │
                    ▼
[Grounded Answer 표시]
    └ 답변 + citations (문서명, 페이지, 청크)
```

### 5.2 문서 편집/다운로드

```
[문서 상세 패널]
    │
    ├──▶ [편집] 클릭
    │         └ POST /api/documents/{id}/edit
    │               └ 인라인 에디터 활성화
    │
    └──▶ [다운로드 ▾] 클릭
              └ GET /api/documents/{id}/download?format=pdf
              └ GET /api/documents/{id}/download?format=txt
              └ GET /api/documents/{id}/download?format=md
              └ GET /api/documents/{id}/download?format=docx
```

## 6. 화면 상태 정의

### 6.1 검색 상태

| 상태 | UI 표시 |
| --- | --- |
| idle | 질문 입력 대기 |
| analyzing | "프롬프트 분석 중..." 로딩 |
| searching | "검색 중..." 탭별 로딩 |
| complete | 결과 표시 |
| error | 에러 메시지, 재시도 버튼 |

### 6.2 문서 상세 상태

| 상태 | UI 표시 |
| --- | --- |
| loading | 스켈레톤 로딩 |
| loaded | 원문/요약/메타 탭 |
| editing | 인라인 에디터 |
| error | 에러 메시지 |

### 6.3 답변 생성 상태

| 상태 | UI 표시 |
| --- | --- |
| idle | "선택 문서 기반 답변 생성" 버튼 |
| generating | "답변 생성 중..." 스트리밍 |
