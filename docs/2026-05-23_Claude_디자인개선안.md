## 2. 핵심 UX 원칙 (DesignUX_BasicRule 기준)

### 2.1 사용자 UX 원칙

1. 쉽게 질문한다.
2. 검색 결과를 먼저 본다.
3. 문서를 직접 선택한다.
4. 선택한 문서만 기반으로 답변을 생성한다.
5. 답변에는 반드시 출처를 표시한다.

### 2.2 관리자 UX 원칙

1. 문서를 스캔한다.
2. 메타데이터를 만든다.
3. 텍스트를 추출한다.
4. Chunk를 만든다.
5. Embedding한다.
6. FAISS를 만든다.
7. Graph와 Wiki를 만든다.
8. Job과 오류를 관리한다.
9. 검색 품질을 검증한다.

## 3. 현재 화면 분석

### 3.1 현재 구현된 기능

이미 구현 완료된 주요 기능은 다음과 같다.

- 검색 모드 Chips UI (하이브리드, 벡터, 전문검색)
- 통합 추천 문서 리스트 (카드형 UI)
- 문서 상세 패널 (슬라이드 인)
- Grounded Answer 패널 (소스 태그 포함)
- Mock 데이터 기반 Demo 버튼

### 3.2 현재 화면의 문제점

| 문제 | 설명 |
|---|---|
| 개발자 설정이 먼저 보인다 | Runtime Target, Backend API, VectorDB, Ollama 등 기술 설정이 초기 화면에 노출 |
| 검색 목적이 불명확하다 | General RAG, Bid Project Search 같은 기술 용어 사용 |
| 문서 선택 흐름이 약하다 | 검색 → 문서 선택 → 답변 생성 흐름이 UI에서 명확하지 않음 |
| Graph/Wiki 설명 부족 | Vector 100%, Graph 0%, Wiki 0% 비율 설정의 의미가 사용자에게 전달되지 않음 |

## 10. API 연동 정리

### 10.1 검색 응답 구조 권장

```json
{
  "recommended_documents": [
    {
      "document_id": "doc_001",
      "title": "K-water 고유 플랫폼 ISP 제안서",
      "file_name": "K-water_ISP_제안서.pdf",
      "document_type": "제안서",
      "document_group": "제안서",
      "document_category": "전략및방법론",
      "organization": "한국수자원공사",
      "project_name": "K-water 고유 플랫폼 구축 전략계획 수립 용역",
      "year": "2024",
      "score": 0.87,
      "reason": "공공기관 ISP와 플랫폼 구축 전략이 유사합니다.",
      "matched_chunks": [],
      "source_mode": "rag"
    }
  ],
  "grounded_answer": {
    "content": "...",
    "citations": [
      {"ref": 1, "document_id": "doc_001", "chunk_id": "chunk_012", "text": "..."}
    ]
  }
}
```

### 10.2 선택 문서 기반 답변 생성 API

- `selected_document_ids` 필수 파라미터
- 문서 미선택 시 400 에러 반환
- 응답에 `citations` 필수 포함

### 10.3 부분 실패 처리

- RAG 성공 + Graph 실패 시 전체 실패 아님
- 추천 문서와 RAG 근거 먼저 표시
- Graph 탭에만 오류 메시지 표시

## 11. 최종 결론

### 11.1 현재 상태

`rag-assistant.html`은 기능은 충분하지만 UX의 중심이 "RAG 설정 콘솔"에 맞춰져 있다.

### 11.2 개선 목표

```
기술 중심 RAG 콘솔
  ↓
문서 기반 AI Agent
  ↓
업무 목적 중심 검색 화면
  ↓
추천 문서 선택
  ↓
선택 문서 기반 답변 생성
  ↓
출처 기반 제안서 초안 생성
```

### 11.3 핵심 개선 3가지

1. **화면 용어를 사용자 업무 중심으로 변경** (일부 완료)
2. **고급 설정을 접고 검색창과 추천 문서를 중심으로 재배치** (진행 필요)
3. **선택 문서 기반 답변 생성 흐름을 명확히 표시** (진행 필요)

이 3가지만 적용해도 현재 페이지는 RAG 테스트 화면에서 문서 기반 AI Agent 화면으로 크게 개선된다.
