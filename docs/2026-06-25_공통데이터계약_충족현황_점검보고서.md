## 4. 주요 미충족 항목

### 4.1 Critical (P0)

1. **dataset_id / snapshot_id 체인**
   - OCR/Parser에서 dataset_id 생성 로직 미연결
   - 파이프라인 전체에 ID 체인 전파 필요

2. **section_id 체계**
   - 문서 내 섹션 식별자 미구현
   - OCR 결과에서 섹션 분리 로직 필요

3. **chunk_id 일관성**
   - FAISS에서 생성되지만 OCR/Parser와 연계 안됨

### 4.2 High (P1)

1. **시맨틱 메타데이터**
   - `topic`, `requirement`, `technology`, `methodology` 필드
   - LLM 기반 추출 또는 규칙 기반 추출 필요

2. **chunk_summary / document_summary**
   - 청크/문서 요약 미구현
   - LLM 또는 추출형 요약 필요

3. **heading_path**
   - 문서 계층 경로 미구현
   - OCR/Parser에서 헤딩 추출 필요

### 4.3 Medium (P2)

1. **organization_type / business_domain**
   - 기관 유형/사업 도메인 분류
   - 온톨로지 기반 매핑 필요

2. **deliverable_type**
   - 산출물 유형 표준화
   - document_category와 구분 필요

## 5. 레이어 간 데이터 흐름 점검

```
OCR/Parser (Step 4)
    ↓ document_id, source_id, file_name, page_no
    ↓ ❌ section_id, chunk_id 미전달
Chunking (Step 5)
    ↓ chunk_id 생성
    ↓ ❌ section_id 미구현
FAISS Build
    ↓ chunk_id, document_id, source_id
    ↓ ⚠️ dataset_id, snapshot_id 부분적
Graph Build
    ↓ document_id, project_name, organization
    ↓ ❌ section_id, chunk_id 미연동
Hybrid Search
    → 결과에 chunk_id, document_id 포함
    → ❌ section_id 미반환
```

## 6. 권장 조치 사항

### 6.1 즉시 조치 (P0)

1. **dataset_id 체인 연결**
   - `document_sources.py`의 `ensure_dataset` 결과를 Step 4에 전달
   - 파이프라인 전체에 dataset_id 전파

2. **section_id 스키마 추가**
   - OCR 결과 테이블에 section_id 컬럼 추가
   - 섹션 분리 로직 구현

3. **FAISS metadata 확장**
   - `build_faiss_index.py`에 누락 필드 추가
   - dataset_id, snapshot_id, file_type, created_at

### 6.2 단기 조치 (P1)

1. **시맨틱 추출 파이프라인**
   - LLM 기반 topic/requirement 추출
   - 키워드 기반 technology/methodology 추출

2. **요약 생성 단계 추가**
   - Step 5 이후 chunk_summary 생성
   - Step 6에서 document_summary 집계

3. **heading_path 추출**
   - OCR 결과에서 헤딩 계층 분석
   - Markdown 변환 시 헤딩 트리 구축

### 6.3 중기 조치 (P2)

1. **온톨로지 매핑**
   - organization_type, business_domain 표준화
   - 동의어 사전 연동

2. **Graph 노드 확장**
   - DocumentSection, Requirement, Topic 노드 추가
   - 청크 단위 연결
