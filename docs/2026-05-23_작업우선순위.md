7. 우선 구현 순서
1순위: 관리자 Sample RAG Builder
목적:
전체 문서 RAG화를 멈추고 폴더별 샘플 30개로 먼저 검색 품질 검증

작업:
- 폴더 스캔
- 샘플 파일 선택
- sample collection 생성
- FAISS Build
- 검색 테스트
2순위: 사용자 검색 화면
목적:
rag-assistant.html에서 샘플 RAG를 바로 검색 가능하게 함

작업:
- 검색창
- 검색 모드
- 필터
- RAG 결과 탭
- 통합 추천 문서
3순위: 선택 문서 기반 답변 생성
목적:
LLM 환각 방지 정책 적용

작업:
- 문서 선택
- generate-answer API
- citations 표시
- 답변 복사
4순위: Graph RAG / LLM Wiki
목적:
단순 벡터 검색을 넘어서 관계 검색과 지식 요약 제공

작업:
- graph nodes/edges 생성
- wiki markdown 생성
- 사용자 검색 탭 연결
5순위: Search Quality
목적:
검색 품질을 지속적으로 개선

작업:
- 테스트 질문 관리
- 검색 결과 비교
- 정답 문서 지정
- 품질 점수화
