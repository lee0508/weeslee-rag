4.1 관리자 페이지의 목적

관리자 페이지는 단순히 파일 목록을 보여주는 화면이 아니다.

관리자 페이지는 다음 작업을 통제해야 한다.

문서 업로드

메타데이터 생성
텍스트 추출
Chunk 확인
Embedding 실행
FAISS 색인 생성
검색 테스트
LLM Wiki 생성
GraphRAG 생성
작업 로그 확인
실패 문서 재처리

4.5 Category Manager 화면

목적: 문서 대분류/세부분류를 관리한다.

권장 분류:

대분류
 ├─ RFP
 ├─ Proposal
 ├─ Final Report
 ├─ Presentation
 ├─ Contract
 ├─ Reference
 ├─ Meeting Note
 └─ Internal Document

세부분류 예시:

Proposal
 ├─ ISP
 ├─ ISMP
 ├─ AI
 ├─ BigData
 ├─ Cloud
 ├─ OCR
 ├─ RAG
 └─ Consulting

주의할 점:

Collection명을 무조건 대분류/세부분류 기준으로 만드는 것은 조심해야 한다.

추천 방식:

FAISS Collection은 너무 잘게 나누지 않는다.
문서 분류는 metadata로 관리한다.
검색 시 metadata filter를 사용한다.

예시:

{
  "collection": "weeslee_documents",
  "metadata": {
    "major_category": "Proposal",
    "minor_category": "ISP",
    "organization": "K-water",
    "project_name": "K-water 고유 플랫폼 구축 전략계획 수립 용역",
    "year": "2024",
    "document_type": "proposal"
  }
}

7단계: 검색 테스트
