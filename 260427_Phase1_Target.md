## 3. 인벤토리 기반 판단

상위 폴더별 집계 결과, 일부 폴더는 1차 PoC 범위를 크게 초과한다.

### 3.1 즉시 제외 대상

다음 폴더는 문서 수와 용량이 너무 커서 1차 전체 복사 대상으로 부적절하다.

1. `[제안실주]` - 17,220 files / 75,691.56 MB
2. `[제안검토]` - 5,431 files / 17,809.46 MB
3. `202212. k-water 데이터허브플랫폼_ISP` - 3,418 files / 18,764.05 MB
4. `202603. AX기반의 차세대 업무 시스템 구축을 위한 ISMP` - 2,882 files
5. `142. 차세대 보건의료빅데이터개방시스템 구축을 위한 ISP` - 2,705 files

이 폴더들은 2차 이상 단계에서 하위 문서유형 또는 특정 연차/버전만 선택하는 방식이 맞다.

### 3.2 1차 적합 방향

1차는 `프로젝트 단위 전체 폴더`보다 `프로젝트 폴더 내부의 문서유형별 대표 파일`을 뽑는 방식이 적합하다.

예:

1. RFP/과업지시서 1건
2. 제안서 1건
3. 착수보고서 1건
4. 중간보고서 1건
5. 최종보고서 1건
6. 발표자료 1건

프로젝트 40~60개에 대해 위 방식으로 표본화하면 300~500건 수준이 된다.

2. 제안서
3. 착수보고서
4. 최종보고서
5. 발표자료

### 4.2 프로젝트 균형

1. 특정 기관에 편중되지 않게 선정
2. 최근 3년 프로젝트를 우선 포함
3. 과거 대표 성공사례 일부 포함
4. ISP/ISMP/AI/빅데이터 등 사업유형을 혼합

1. 디지털 PDF 포함
2. 스캔 PDF 포함
3. PPT/PPTX 포함
4. DOC/DOCX 포함

### 4.4 보안/운영 원칙

1. 원본 이동 금지
2. snapshot 복사만 수행
3. 개인 메모/임시본/중복본은 1차 제외
4. 대용량 압축파일은 1차 제외

## 5. 1차 추천 표본 전략

1차에서는 아래 전략이 적절하다.

### 전략 A: 프로젝트 50개 x 문서 6종

1. 프로젝트 50개 선정
2. 프로젝트별 대표 문서 6종 선택
3. 목표 수량 약 300건

장점:

1. 사업 흐름 전체를 보기 좋다
2. 그래프 관계 생성에 유리하다

### 전략 B: 최근 프로젝트 30개 + 과거 대표 프로젝트 20개

1. 최근 프로젝트 30개
2. 과거 대표 프로젝트 20개
3. 프로젝트별 6~8건 문서 선택

장점:

1. 최신 문서 양식과 과거 문서 재사용 패턴을 함께 학습할 수 있다

본 과제에는 `전략 B`를 권장한다.
