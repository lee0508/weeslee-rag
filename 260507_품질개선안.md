왜 이렇게 가야 하는가?

현재 metadata에서 확실한 값은 다음입니다.

document_id
project_name
folder_year
category
source_path
extension

반대로 아직 불확실한 값은:

organization
methodology
requirement
deliverable

입니다.

그래서 1차부터 Organization을 넣으면 그래프가 그럴듯해 보이지만 데이터 품질이 흔들릴 수 있습니다.

1차 그래프 구조
Node
Project
Document
Category
Edge
Project → has_document → Document
Document → has_category → Category
Document → related_sequence → Document
1차 Edge 규칙

같은 project_name 안에서 category 순서를 기준으로 연결합니다.

rfp
→ proposal
→ kickoff
→ presentation
→ final_report

단, 프로젝트마다 문서가 없을 수 있으므로 존재하는 것만 연결합니다.
