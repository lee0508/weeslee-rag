## 0. weeslee-rag의 본질적 목표

weeslee-rag는 단순히 문서를 업로드하고 검색하는 시스템이 아니다.

본질적인 목표는 다음과 같다.

1. 회사 내부 문서, 제안서, 완료보고서, 발표자료, RFP, 사업계획서 등을 중앙화한다.
2. 문서를 자동으로 분류하고 메타데이터를 생성한다.
3. 문서 내용을 Chunk 단위로 분리하고 Embedding 한다.
4. FAISS 또는 Chroma 같은 VectorDB에 저장한다.
5. 사용자가 신규 사업명, RFP, 키워드, 기관명으로 검색하면 관련 문서를 찾아준다.
6. 단순 검색 결과가 아니라 “왜 이 문서가 관련 있는지” 설명해준다.
7. 장기적으로는 LLM Wiki와 GraphRAG를 통해 문서 간 관계, 사업 간 관계, 기관 간 관계까지 구조화한다.
8. 최종적으로 제안서 작성, 사업 분석, 유사 사업 검색, 회사 레퍼런스 추천까지 지원한다.

즉, weeslee-rag는 다음 4단계로 발전해야 한다.

1단계: 문서 중앙화  
2단계: RAG 기반 문서 검색  
3단계: LLM Wiki 기반 지식 정리  
4단계: GraphRAG 기반 관계 추론 및 제안서 작성 지원

# 1. LangChain 이란?

## 1.1 정의

LangChain은 LLM 기반 애플리케이션을 만들기 위한 오픈소스 프레임워크이다.  
공식 GitHub 설명 기준으로 LangChain은 LLM-powered application과 Agent를 만들기 위한 프레임워크이며, 여러 컴포넌트와 외부 도구를 연결할 수 있게 해준다.  
LangChain은 Python과 JavaScript/TypeScript 기반 개발을 지원한다. :contentReference[oaicite:0]{index=0}

쉽게 말하면 LangChain은 다음 작업을 쉽게 해주는 도구이다.

- LLM 호출
- Prompt 관리
- 문서 로딩
- 텍스트 분할
- Embedding 생성
- VectorDB 검색
- 검색 결과를 LLM에 전달
- 답변 생성
- Agent 구성
- 외부 API, DB, 파일 시스템 연결

## 1.2 weeslee-rag에서 LangChain이 필요한 이유

weeslee-rag는 단순 챗봇이 아니라 문서 기반 검색 시스템이다.

따라서 다음 기능이 필요하다.

- PDF, HWP, DOCX, TXT, MD, CSV 문서 읽기
- 문서를 일정 크기로 Chunking
- Chunk를 Embedding
- VectorDB에 저장
- 검색어를 Embedding
- 유사 문서 검색
- 검색 결과를 LLM에 넣어서 답변 생성
- 출처 문서 표시
- 관리자 페이지에서 색인 상태 확인

이 흐름을 직접 모두 구현할 수도 있지만, LangChain을 사용하면 문서 처리, Retrieval, Prompt, LLM 연결 구조를 표준화할 수 있다.
