## 변경 내용

### assemble_rag_response.py - build_prompt()

#### 시스템 프롬프트 개선

```
## 핵심 원칙 (Grounded Answer)
1. 반드시 아래 검색 결과에 있는 근거만 사용하라. 추측하거나 과장하지 마라.
2. 검색 결과에 없는 내용은 답변하지 마라.
3. 근거가 부족하면 '관련 문서에서 명확한 근거를 찾지 못했습니다'라고 명시하라.
4. 답변 문장에는 반드시 Citation을 붙여라. 형식: [1], [2], [3]
```

#### 출력 형식 정의

```
### 요약 답변
(질의에 대한 답변. 각 문장에 Citation 번호를 붙인다.)

### 근거 문서
[1] 파일명 - 경로
[2] 파일명 - 경로
...

### 활용 포인트
(제안서 작성에 바로 활용할 수 있는 포인트 1-3개)
```

#### 검색 결과 형식 개선

- 문서마다 Citation 번호 부여: `[1] 파일명`
- 경로 표시: `경로: {display_path}`
- 분류/카테고리 표시
- 점수/히트 수 간결화

### 예상 출력 예시

```
### 요약 답변
이 사업은 AI 기반 정보화전략계획(ISP) 수립을 목표로 합니다. [1]
전략 및 방법론은 데이터 중심 아키텍처 설계를 강조합니다. [2]

### 근거 문서
[1] RFP_AI 기반 e-감사시스템 재구축 ISP 컨설팅.hwp - 01. RFP/
[2] 전략및방법론_AI 기반 e-감사시스템 재구축 ISP 컨설팅.pptx - 02. 제안서/01. 전략및방법론/

### 활용 포인트
1. ISP 수립 방법론 참조
2. 데이터 아키텍처 설계 패턴 적용
3. 단계별 추진 일정 템플릿 활용
```
