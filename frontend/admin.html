관리자 작업 흐름

이 화면은 회사 문서를 사용자 검색 화면에서 활용할 수 있도록 원본 문서 등록부터 데이터셋 생성, 지식그래프 생성, 위키 생성, 검색 품질 확인, 운영 반영까지 처리하는 관리자 콘솔입니다.

  1. Document Source : 원본 문서 폴더를 등록합니다.
  2. Dataset Builder : 문서를 분석하여 검색용 FAISS 데이터셋을 생성합니다.
  3. Knowledge Graph : 문서 간 관계와 키워드 관계를 그래프로 생성합니다.
  4. LLM Wiki : 문서 내용을 요약하여 AI 참고용 위키를 생성합니다.
  5. Publish : 검증된 Dataset/Snapshot을 사용자 검색 화면에 반영합니다.

Source / 문서 분류 기준

  • Source ID: 원본 문서 저장소 ID, 예: rag_source
  • 문서 대분류(document_group): Source 아래의 1차 분류, 예: RFP, 제안서, 산출물
  • 문서 세부분류(section_type): 대분류 아래의 세부 분류, 예: 전략및방법론, 프로젝트관리, 연구과제, 이행계획
  • 문서 ID(document_id): 현재 DB/API 기준 숫자형 파일 고유 ID
  • 문서 UID(document_uid): source_id + relative_path 기반 내부 추적 ID

    문서 메타데이터와 청크 데이터를 기반으로 Knowledge Graph를 생성합니다.
    
        
    
          안내: 현재 Knowledge Graph는 JSON Graph 파일 기반으로 운영됩니다.
          Neo4j는 향후 확장 옵션이며 현재 기본 저장소가 아닙니다.

              SIMILAR_TO 관계 생성 기준
