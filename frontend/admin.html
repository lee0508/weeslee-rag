시스템 작동 원리

RAG(Retrieval-Augmented Generation)는 문서를 벡터화하여 저장하고, 사용자 질문과 유사한 문서를 검색한 후 LLM이 해당 문서를 참조하여 답변을 생성하는 방식입니다.
1

문서 수집

네트워크 드라이브나 로컬 폴더에서 PDF, HWP, DOCX 등 문서를 스캔하고 텍스트를 추출합니다.

2

벡터 인덱싱

추출된 텍스트를 청크로 분할하고 임베딩하여 FAISS 벡터 인덱스에 저장합니다.

3

검색 및 생성

사용자 질문과 유사한 문서를 검색하고, LLM이 해당 문서를 참조하여 근거 기반 답변을 생성합니다.

정상 작동을 위한 순차 작업

시스템이 정상 작동하려면 아래 순서대로 작업을 완료해야 합니다. 각 단계의 상태를 확인하고 필요한 작업을 진행하세요.

1 소스 등록 - 문서가 저장된 폴더 경로를 등록합니다. 확인 중...
2 파일 스캔 - 등록된 소스에서 문서 파일을 스캔합니다. 확인 중...
3 메타데이터 생성 - 문서별 분류, 기관명, 사업명 등 메타데이터를 추출합니다. 확인 중...
4 FAISS 인덱스 생성 - 문서를 벡터화하여 검색 인덱스를 생성합니다. 확인 중...
5 Graph/Wiki 생성 - 문서 간 관계 그래프와 프로젝트별 Wiki를 생성합니다. 확인 중...
6 검색 테스트 - 검색 품질을 테스트하고 결과를 확인합니다. 확인 중...
