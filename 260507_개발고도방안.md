현재 상태:
RAG 검색 화면 + FAISS 관리자 화면 + Graph View가 분리되어 구현 중

다음 방향:
“입찰 사업명 기반 문서 추천 시스템”으로 기능을 묶어야 함
1. 전체 구조 판단

main.py는 현재 필요한 라우터를 거의 다 포함하고 있습니다.

app.include_router(rag_router, prefix="/api", tags=["RAG"])
app.include_router(files_router, prefix="/api", tags=["Files"])
app.include_router(faiss_admin_router, prefix="/api", tags=["FAISS Admin"])
app.include_router(graph_router, prefix="/api", tags=["Graph"])

즉 백엔드는 이제 단순 RAG API가 아니라:

RAG 검색
파일 다운로드
FAISS 운영관리
Graph View
관리자 기능

까지 확장된 상태입니다.

2. admin.html 분석

admin.html은 이전 단순 문서 처리 화면에서 벗어나 FAISS Index Management Console로 잘 전환되었습니다.

현재 포함된 기능:

1. 활성 인덱스 상태
2. Ollama 상태
3. 카테고리 인덱스 상태
4. 파이프라인 실행
5. 인덱스 목록
6. 잡 이력
7. 검색 품질 벤치마크
8. Graph View

특히 Graph View가 이미 Cytoscape.js 기반으로 들어가 있고, /graph/summary, /graph/projects, /graph/project/{project_name} API를 호출하는 구조도 잡혀 있습니다.

이건 매우 좋은 방향입니다.

3. rag-assistant.html 분석

rag-assistant.html은 실제 사용자용 검색 화면입니다.

현재 핵심 기능:

1. Query 입력
2. Top K 설정
3. Top Docs 설정
4. Max Chunks / Doc 설정
5. Category 선택
6. Answer Mode 선택
   - Ollama Generate
   - Search Only
7. Model 선택
8. 추천 문서 표시
9. 답변 생성
10. 파일 다운로드 / 경로 복사

특히 /rag/query로 아래 값을 넘기는 구조가 이미 잘 잡혀 있습니다.

{
  query,
  top_k: topK,
  top_docs: topDocs,
  max_chunks_per_doc: maxChunks,
  answer_provider: answerProvider,
  answer_model: answerProvider === 'none' ? '' : answerModel,
  category
}

즉 현재 시스템은 이미 검색 품질 튜닝 가능한 RAG 워크스페이스입니다.

4. 현재 가장 중요한 문제

현재 기능은 많아졌지만, 사용 시나리오가 아직 조금 분산되어 있습니다.

지금 화면은 개발자용으로는 좋지만, 실제 회사 사용자는 이렇게 묻습니다.

“나라장터 사업명을 넣으면 어떤 기존 문서를 참고해야 하나?”

따라서 다음 개발 방향은 이것입니다.

RAG Assistant를 “입찰 사업명 분석 화면”으로 고도화

7. admin.html의 다음 작업

admin.html은 지금 방향이 좋습니다. 다음은 안정화 작업입니다.

1. active index 상태와 rag-assistant 사용 인덱스 일치 확인
2. category index 상태가 실제 검색과 일치하는지 확인
3. Graph Build 실행 후 프로젝트 목록 정상 표시 확인
4. Benchmark 결과를 날짜별 저장
5. Ollama timeout / search-only mode 분리

특히 Benchmark는 반드시 search_only 기준으로 고정해야 합니다. Ollama 생성까지 넣으면 timeout 때문에 검색 품질 판단이 흐려집니다.

8. Graph View 다음 방향

현재 Graph View는 1차 구현으로 충분합니다.

다음 단계는:

1. Project → Document 관계 안정화
2. Document → Category 관계 표시
3. RFP → Proposal → Kickoff → Presentation → Final Report 순서 표시
4. 노드 클릭 시 rag-assistant 검색으로 연결

예를 들어 Graph View에서 문서를 클릭하면:

이 문서와 유사한 문서 검색
이 프로젝트 기준 RAG 검색
파일 다운로드

버튼이 있으면 좋습니다.

9. 파일 다운로드 기능은 유지

rag-assistant.html에는 이미 /files/download?path= 방식이 들어가 있습니다.

이 기능은 매우 중요합니다.

왜냐하면 실제 사용자는 검색 결과만 보는 것이 아니라:

가 목적이기 때문입니다.
