## 0. 프로젝트 개요 및 목적

### 0.1 개발 목적

**핵심 목적:**
> 회사의 컨설팅 문서들을 **RAG 시스템**을 통해 컬렉션화하여 **VectorDB**에 저장하고,
> **프롬프트**를 통해 관련 문서를 검색하여 **유사 문서 또는 새로운 문서를 생성/저장**하는 시스템

### 0.3 주요 사용 시나리오

| 시나리오 | 설명 | 예시 |

## 1. 상단 네비게이션 메뉴 기능 정의

### 1.1 현재 UI 구조

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ ⬡ PromptoRAG   [🏠 홈] [📚 프롬프트 라이브러리] [▶ 새 작업 실행] [📋 실행 이력] [⚙ 관리자]  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 메뉴별 기능 정의

| 메뉴 | 기능 | 동작 방식 | 비고 |
|------|------|-----------|------|
| **🏠 홈** | 대시보드 | 별도 페이지 이동 | 최근 작업, 통계 요약 |
| **📚 프롬프트 라이브러리** | 템플릿 목록 | 별도 페이지 이동 | 템플릿 검색/관리 |
| **▶ 새 작업 실행** | AI 생성 실행 | **버튼 동작** | 하단 "AI 생성 실행"과 동일 |
| **📋 실행 이력** | 작업 이력 목록 | 별도 페이지 이동 | 과거 실행 결과 조회 |
| **⚙ 관리자** | 관리자 기능 | 모달 또는 페이지 | 템플릿 승인, 지식원 관리 |

### 1.3 "▶ 새 작업 실행" 메뉴 상세

**핵심 요구 사항:**
> 상단 "새 작업 실행" 메뉴 기능은 중앙 페이지 하단에 있는 "AI 생성 실행" 버튼 기능과 **동일**합니다.

#### 동작 로직

```
사용자가 "▶ 새 작업 실행" 클릭
        │
        ▼
┌───────────────────────────────────────┐
│ 필수 입력값 검증                       │
│ • 템플릿 선택 여부                     │
│ • 필수 파라미터 입력 여부              │
│ • 지식원 선택 여부                     │
└───────────────────────────────────────┘
        │
        ├── 검증 실패 → 경고 메시지 표시
        │              "필수 입력값을 확인해주세요"
        │              해당 Step으로 포커스 이동
        │
        └── 검증 성공 → AI 생성 실행 시작
                       (runGenerate() 함수 호출)
```

#### 현재 하단 버튼 코드 (참고)

```html
<!-- 하단 실행 바 -->
<div class="run-bar">
  <div class="run-status">
    <span id="run-status-text">⚡ 입력값 설정 후 실행 가능합니다</span>
  </div>
  <span class="token-counter">예상 토큰: ~1,240</span>
  <button class="btn btn-outline btn-sm" onclick="openModal('modal-settings')">⚙ 모델 설정</button>
  <button class="btn btn-amber btn-lg" id="run-btn" onclick="runGenerate()">
    ▶ AI 생성 실행
  </button>
</div>
```

#### 구현 방향

| 방식 | 설명 | 장단점 |
|------|------|--------|
| **방식 A: 동일 함수 호출** | 상단 메뉴 클릭 시 `runGenerate()` 함수 직접 호출 | 단순, 일관성 유지 |
| **방식 B: 하단 버튼 트리거** | 상단 메뉴 클릭 시 하단 버튼 클릭 이벤트 발생 | 버튼 상태(disabled 등) 자동 반영 |

**권장:** 방식 A (동일 함수 호출)

```javascript
// 상단 네비게이션 메뉴
<button class="topnav-tab" onclick="runGenerate()">▶ 새 작업 실행</button>
```

### 2.2 프롬프트 검색 영역

#### 2.2.1 검색 입력창

| 항목 | 요구 사항 |
|------|-----------|
| 검색 대상 | 템플릿 이름, 설명, 태그 |
| 실시간 필터링 | 입력 시 즉시 목록 필터링 |
| 플레이스홀더 | "프롬프트 검색..." |

#### 2.2.2 카테고리 필터 칩

| 카테고리 | 설명 | 문서 메타태그 값 |
|----------|------|------------------|
| **전체** | 모든 템플릿 표시 | - |
| **보고서** | 보고서 작성용 템플릿 | `category: report` |
| **질의응답** | Q&A, 법령해석 등 | `category: qa` |
| **요약** | 문서 요약용 템플릿 | `category: summary` |
| **분석** | 비교분석, 벤치마킹 등 | `category: analysis` |

**핵심 요구 사항:**
> 카테고리 필터 칩은 **문서(템플릿)의 메타태그**와 동일해야 합니다.

**동작 규칙:**

| 사용자 액션 | 결과 |
|-------------|------|
| "전체" 클릭 | 모든 템플릿 표시, 다른 칩 선택 해제 |
| 개별 카테고리 클릭 | 해당 카테고리만 필터링 |
| 다중 선택 | 불가 (단일 선택) |

### 2.3 즐겨찾기 영역

**핵심 요구 사항:**
> 새 템플릿 작성 시 **"즐겨찾기" 체크박스**가 있어야 하며, 체크된 템플릿만 이 영역에 표시됩니다.

#### 표시 규칙

| 항목 | 요구 사항 |
|------|-----------|
| 표시 조건 | 템플릿의 `is_favorite = true` |
| 최대 표시 개수 | **4개** |
| 초과 시 동작 | **Y축 스크롤** 활성화 |
| 정렬 기준 | 최근 사용순 또는 등록순 |
| 빈 상태 | "즐겨찾기한 템플릿이 없습니다" |

#### 데이터 모델

```sql
-- prompts 테이블에 필드 추가
ALTER TABLE prompts ADD COLUMN is_favorite BOOLEAN DEFAULT FALSE;

-- 또는 별도 테이블 (사용자별 즐겨찾기)
CREATE TABLE user_favorites (
    id          INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT NOT NULL,
    prompt_id   INT NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY (user_id, prompt_id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (prompt_id) REFERENCES prompts(id)
);
```

### 2.4 조직 표준 템플릿 영역

**핵심 요구 사항:**
> 새 템플릿 작성 시 **"조직 표준 템플릿" 체크박스**가 있어야 하며, 체크된 템플릿만 이 영역에 표시됩니다.

#### 표시 규칙

| 항목 | 요구 사항 |
|------|-----------|
| 표시 조건 | 템플릿의 `is_org_standard = true` AND `status = 'approved'` |
| 최대 표시 개수 | **4개** |
| 초과 시 동작 | **Y축 스크롤** 활성화 |
| 정렬 기준 | 사용 횟수 많은순 또는 최신순 |
| 권한 | 관리자 승인 필요 |

#### 데이터 모델

```sql
-- prompts 테이블에 필드 추가
ALTER TABLE prompts ADD COLUMN is_org_standard BOOLEAN DEFAULT FALSE;

-- 승인 프로세스 필요
-- status: 'draft' → 'review' → 'approved'
-- is_org_standard = true이고 status = 'approved'인 경우에만 표시
```

#### 승인 프로세스

```
사용자가 "조직 표준 템플릿" 체크 후 저장
        │
        ▼
┌───────────────────────────────────────┐
│ 상태: review (검토중)                  │
│ 관리자 승인 대기                       │
└───────────────────────────────────────┘
        │
        ├── 관리자 승인 → status = 'approved'
        │               → 조직 표준 템플릿 영역에 표시
        │
        └── 관리자 반려 → status = 'rejected'
                        → 개인 저장 영역으로 이동
```

### 2.5 개인 저장 영역

**핵심 요구 사항:**
> 새 템플릿 작성 시 **"개인 저장" 체크박스**가 있어야 하며, 체크된 템플릿만 이 영역에 표시됩니다.

#### 표시 규칙

| 항목 | 요구 사항 |
|------|-----------|
| 표시 조건 | 템플릿의 `is_personal = true` AND `creator_id = 현재 사용자` |
| 최대 표시 개수 | **4개** |
| 초과 시 동작 | **Y축 스크롤** 활성화 |
| 정렬 기준 | 최근 수정순 |
| 권한 | 본인만 조회/수정/삭제 가능 |

#### 데이터 모델

```sql
-- prompts 테이블에 필드 추가
ALTER TABLE prompts ADD COLUMN is_personal BOOLEAN DEFAULT FALSE;

-- 조회 조건
WHERE is_personal = TRUE
  AND creator_id = :current_user_id
```

### 2.6 새 템플릿 만들기 모달 수정

**핵심 요구 사항:**
> 새 템플릿 만들기 모달에 **3개의 체크박스**가 추가되어야 합니다.

### 3.4 Step 1, 2 자동 선택 동작

**핵심 요구 사항:**
> 좌측 사이드바에서 템플릿을 선택하면 **Step 1, 2가 자동으로 완료** 처리됩니다.

#### 동작 흐름

```
좌측 사이드바에서 템플릿 클릭
        │
        ▼
┌───────────────────────────────────────┐
│ 1. Step 1 "업무선택" 자동 완료 (✓)    │
│    → 템플릿의 카테고리 자동 선택       │
│                                       │
│ 2. Step 2 "프롬프트선택" 자동 완료 (✓)│
│    → 선택한 템플릿 활성화              │
│                                       │
│ 3. Step 3 "입력값 설정" 활성화        │
│    → 중앙 영역으로 자동 이동           │
│                                       │
│ 4. 중앙 영역에 템플릿 내용 표시        │
│    → 템플릿 헤더 + 입력 폼 렌더링      │
└───────────────────────────────────────┘
```

#### 구현 코드

```javascript
function selectTemplate(element, templateId) {
  // 1. 템플릿 데이터 로드
  const template = loadTemplate(templateId);

  // 2. Step 1 자동 완료: 카테고리 자동 선택
  const category = template.category;
  selectCategoryChip(category);  // 카테고리 필터 칩 활성화
  setStepComplete(1);

  // 3. Step 2 자동 완료: 템플릿 선택 완료
  setStepComplete(2);

  // 4. Step 3 활성화: 입력값 설정으로 이동
  setCurrentStep(3);

  // 5. 중앙 영역에 템플릿 내용 표시
  renderTemplateHeader(template);
  renderInputParameters(template.variables);

  // 6. 좌측 사이드바 활성 상태 업데이트
  updateSidebarActiveState(element);

  // 7. 입력 파라미터 영역으로 스크롤
  scrollToElement('#input-params-section');
}
```

### 4.6 AI 생성 실행 API

#### Ollama Generate API 호출

```javascript
async function generateWithOllama(prompt, options) {
  const response = await fetch('http://localhost:11434/api/generate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: options.model,        // "llama3:8b"
      prompt: prompt,
      stream: true,                // SSE 스트리밍
      options: {
        temperature: options.temperature,
        top_p: options.top_p,
        top_k: options.top_k,
        num_predict: options.max_tokens
      }
    })
  });

  // 스트리밍 응답 처리
  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n').filter(line => line.trim());

    for (const line of lines) {
      const json = JSON.parse(line);
      appendToResult(json.response);  // UI에 실시간 출력
    }
  }
}
```

### 6.9 데이터베이스 스키마

```sql
-- 문서 테이블
CREATE TABLE documents (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    collection_id   INT NOT NULL,
    filename        VARCHAR(255) NOT NULL,
    original_path   VARCHAR(500),
    file_type       ENUM('ppt', 'pptx', 'doc', 'docx', 'hwp', 'hwpx', 'pdf', 'xlsx'),
    file_size       BIGINT,
    status          ENUM('pending', 'extracting', 'chunking', 'embedding', 'completed', 'failed'),
    error_message   TEXT,
    chunk_count     INT DEFAULT 0,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (collection_id) REFERENCES collections(id)
);

-- 컬렉션 테이블
CREATE TABLE collections (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    name            VARCHAR(100) NOT NULL UNIQUE,
    description     TEXT,
    is_system       BOOLEAN DEFAULT FALSE,  -- "전체" 컬렉션은 시스템 컬렉션
    document_count  INT DEFAULT 0,
    vector_count    INT DEFAULT 0,
    storage_size    BIGINT DEFAULT 0,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 문서 청크 테이블 (메타데이터만, 벡터는 VectorDB)
CREATE TABLE document_chunks (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    document_id     INT NOT NULL,
    chunk_index     INT NOT NULL,
    content         TEXT NOT NULL,
    token_count     INT,
    page_number     INT,
    vector_id       VARCHAR(100),  -- Chroma에서의 ID
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE
);

-- 처리 로그 테이블
CREATE TABLE processing_logs (
    id              INT PRIMARY KEY AUTO_INCREMENT,
    document_id     INT NOT NULL,
    status          VARCHAR(50),
    message         TEXT,
    progress        INT,  -- 0-100
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (document_id) REFERENCES documents(id) ON DELETE CASCADE
);
```
