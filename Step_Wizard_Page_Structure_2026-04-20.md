## 현재 시안 분석: 단일 페이지형 Wizard

### 핵심 발견

현재 HTML은 **페이지 전환 없이 단일 화면에서 모든 단계를 처리**하는 구조입니다.

```
┌──────────────┬─────────────────────────────────┬──────────────┐
│   좌측       │            중앙                  │    우측      │
│  사이드바     │                                 │    패널      │
│  (270px)     │         (flex: 1)               │   (320px)    │
├──────────────┼─────────────────────────────────┼──────────────┤
│              │  ┌─────────────────────────┐    │              │
│  🔍 검색     │  │ [Step Wizard 진행 바]   │    │  📚 지식원   │
│              │  └─────────────────────────┘    │     선택     │
│  ─────────   │                                 │   (Step 4)   │
│              │  ┌─────────────────────────┐    │              │
│  ⭐ 즐겨찾기  │  │  중앙 컨텐츠 영역       │    │  ─────────   │
│              │  │  (Step별 변경)          │    │              │
│  🏛 조직표준  │  │                         │    │  🔧 검색조건  │
│   (Step 2)   │  │  Step 1: 업무선택 카드   │    │              │
│              │  │  Step 2: 템플릿 상세     │    │  ─────────   │
│  👤 개인저장  │  │  Step 3: 입력 파라미터   │    │              │
│              │  │  Step 4: 지식원 확장     │    │  📌 근거문서  │
│  ─────────   │  │  Step 5: AI 결과        │    │   (Step 5)   │
│              │  │                         │    │              │
│  🕐 최근사용  │  └─────────────────────────┘    │  ─────────   │
│              │                                 │              │
│              │  ┌─────────────────────────┐    │  ⚠️ 주의알림  │
│              │  │ [실행 바 - 항상 표시]    │    │              │
│              │  │  상태 | 토큰 | 실행버튼  │    │  📊 통계     │
│              │  └─────────────────────────┘    │              │
└──────────────┴─────────────────────────────────┴──────────────┘
```

### 레이아웃 특징

| 영역 | 상시 표시 | Step별 역할 |
|------|----------|------------|
| **좌측 사이드바** | ✅ 항상 표시 | Step 2에서 템플릿 선택 |
| **중앙 컨텐츠** | ✅ 항상 표시 | Step별로 내용 변경 |
| **우측 패널** | ✅ 항상 표시 | Step 4에서 지식원 선택, Step 5에서 근거문서 |
| **하단 실행 바** | ✅ 항상 표시 | 모든 단계에서 실행 가능 |

```

  중앙 영역      좌측+중앙       중앙 영역      우측→중앙 확장   중앙+우측
  카드 선택      목록+상세       현재 HTML      지식원 상세      결과+근거
```

## Step 1: 업무선택 (문서 유형 선택)

### 목적
사용자가 **어떤 종류의 문서를 만들 것인지** 대분류를 선택합니다.

### 화면 변화

| 영역 | 변화 |
|------|------|
| **좌측 사이드바** | 그대로 유지 (또는 접힘) |
| **중앙 영역** | 🔄 **카테고리 선택 카드 표시** |
| **우측 패널** | 그대로 유지 (또는 접힘) |

### 상호작용

| 액션 | 결과 |

| 최근 작업 이어하기 | Step 3로 바로 이동 (이전 입력값 복원) |
| 전체 템플릿 보기 | Step 2로 이동, 필터 없이 전체 목록 |

## Step 2: 템플릿선택 (작업 템플릿 선택)

### 목적
**좌측 사이드바**에서 템플릿을 선택하고, **중앙 영역**에서 선택한 템플릿의 상세 정보를 확인합니다.

### 화면 변화

| 영역 | 변화 |
|------|------|
| **좌측 사이드바** | 🔄 **활성화** - 템플릿 목록 탐색/선택 |
| **중앙 영역** | 🔄 **선택한 템플릿 상세 정보 표시** |
| **우측 패널** | 그대로 유지 (지식원 선택 대기) |

## Step 3: 입력값 설정 (파라미터 입력)

### 목적
선택한 템플릿의 **변수(파라미터)를 입력**하고 **추가 요청**을 작성합니다.
**현재 HTML의 메인 화면을 그대로 활용**합니다.

### 화면 변화

| 영역 | 변화 |
|------|------|
| **좌측 사이드바** | 그대로 유지 (템플릿 변경 가능) |
| **중앙 영역** | 🔄 **입력 폼 + 추가 요청 영역 표시** |
| **우측 패널** | 그대로 유지 (지식원 선택 가능) |

- **우측**: 지식원/검색 조건 사전 설정

이는 Step 4를 건너뛰고 바로 실행할 수 있는 **빠른 작업 흐름**을 지원합니다.

### 디자인 확인 사항

- [ ] 필드 유형별 컴포넌트 (text, select, textarea, date 등)
- [ ] 필수값 미입력 시 유효성 검사 UI
- [ ] 미리보기 영역 높이/스크롤
- [ ] 필드 개수 많을 때 스크롤 vs 아코디언
- [ ] 좌측/우측 패널 접기 버튼 필요 여부 (중앙 영역 확장)

## 단일 페이지형 Wizard의 핵심 장점

### 기존 다중 페이지 방식 vs 현재 단일 페이지 방식

| 구분 | 다중 페이지 | 단일 페이지 (현재 시안) |
|------|-----------|----------------------|
| **화면 전환** | 매 단계 페이지 이동 | 중앙 영역만 변경 |
| **컨텍스트 유지** | 단계 이동 시 정보 소실 | 모든 정보 상시 표시 |
| **빠른 수정** | 이전 단계로 돌아가야 함 | 좌측/우측 패널에서 즉시 수정 |
| **숙련자 효율** | 단계별 이동 필수 | 원하는 영역 직접 접근 |

### 핵심 설계 원칙

1. **3-패널 상시 표시**: 좌측(템플릿) + 중앙(작업) + 우측(지식원)
2. **중앙 영역만 Step별 변경**: 레이아웃 안정성 유지
3. **하단 실행 바 항상 표시**: 어느 단계에서든 실행 가능
4. **패널 간 상호작용**: Step 3에서 우측 지식원 선택 등 크로스 패널 작업

### 디자인 확인 사항 (전체)

- [ ] 3-패널 레이아웃 최종 확정 (너비 비율)
- [ ] 패널 접기/확장 기능 필요 여부
- [ ] Step 진행 표시 (상단 프로그레스 바) 디자인
- [ ] 모바일/태블릿 대응 시 레이아웃 변경 방안
